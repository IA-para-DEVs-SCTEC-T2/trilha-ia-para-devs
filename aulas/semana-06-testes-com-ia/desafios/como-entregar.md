# Como Entregar

## Formato de entrega

A entrega deve ser feita via Pull Request no repositorio da disciplina ou conforme orientacao do professor.

## Estrutura esperada da entrega

```
atividades/semana-06/
└── seu-nome-ou-grupo/
    ├── tests/
    │   └── test_nome_do_modulo.py
    ├── evidencias/
    │   └── resultado-execucao.png (ou .txt)
    ├── prompts-usados.md
    └── README.md
```

## Passos para entrega

1. Crie uma branch com o padrao: `feat/semana-06-{seu-nome}`.
2. Adicione seus arquivos na pasta `atividades/semana-06/`.
3. Faca commit com mensagem descritiva: `feat: adiciona testes semana 06 - {seu-nome}`.
4. Faca push da branch.
5. Abra um Pull Request com titulo e descricao claros.
6. Aguarde a revisao.

## O que incluir na entrega

- Arquivos de teste com os cenarios implementados.
- Evidencias de execucao (print do terminal ou arquivo de log).
- Arquivo `prompts-usados.md` com os prompts utilizados.
- README explicando o que foi feito e como executar os testes.

## Prazo

Conforme informado pelo professor em aula.
