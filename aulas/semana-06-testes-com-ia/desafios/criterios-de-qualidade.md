# Criterios de Qualidade - Desafios de Testes

## Criterios obrigatorios

- Os testes devem executar sem erros de sintaxe ou importacao.
- Os testes devem cobrir pelo menos um cenario positivo e um negativo.
- Os nomes dos testes devem ser descritivos.
- As evidencias de execucao devem estar incluidas na entrega.
- O arquivo `prompts-usados.md` deve estar presente.

## Criterios de qualidade dos testes

| Criterio | Esperado |
|----------|----------|
| Nomeacao | Nomes descritivos que explicam o que o teste verifica |
| Cobertura | Cenarios positivos, negativos, limites e edge cases |
| Assertivas | Assertivas especificas, nao apenas `assert True` |
| Isolamento | Cada teste verifica uma coisa especifica |
| Legibilidade | Codigo limpo e facil de entender |

## Criterios de qualidade da documentacao

| Criterio | Esperado |
|----------|----------|
| README | Explica o que foi feito e como executar |
| Prompts | Prompts documentados com resultado e ajustes |
| Evidencias | Print ou log da execucao dos testes |

## Niveis de avaliacao

**Basico:** Testes funcionam, cobrem o cenario principal, evidencias incluidas.

**Intermediario:** Testes cobrem cenarios positivos e negativos, nomes descritivos, prompts documentados.

**Avancado:** Cobertura ampla incluindo limites e edge cases, testes bem estruturados, uso critico e documentado de IA, README claro.
