# Semana 05 - Arquiteturas, Mockups e Fluxo GitHub

## Objetivo da semana

Apresentar como usar IA para apoiar a fase de planejamento e documentacao inicial de um projeto de software, cobrindo desde a criacao de User Stories ate a organizacao do repositorio com Issues, branches e Pull Requests.

## Conteudos principais

- Criacao de User Stories com formato Como / Quero / Para
- Criterios de aceitacao em BDD (Given / When / Then)
- Construcao de backlog
- Documentacao inicial com apoio de IA
- Organizacao de repositorio no GitHub
- Criacao e gestao de branches
- Abertura e revisao de Pull Requests
- Convencoes de commits e branches geradas com apoio de IA

## Como estudar o material

1. Leia os slides para entender o fluxo completo.
2. Acesse `hands-on/01-kiro-github-issues-prs-automacao/` e leia o README do projeto.
3. Leia o `roteiro.md` para entender a sequencia praticada em aula.
4. Consulte `fluxo-github.md` para revisar o fluxo de Issues, branches e PRs.
5. Consulte `comandos.md` para referencia rapida dos comandos usados.
6. Leia o manual disponivel em `materiais/` para aprofundamento.

## Relacao com o mini-projeto

Esta semana e diretamente aplicada ao mini-projeto. Voce deve:

- Criar as User Stories do seu projeto usando o formato aprendido.
- Organizar o backlog no GitHub Projects ou como Issues.
- Criar branches para cada funcionalidade.
- Abrir Pull Requests com descricao adequada.
- Consulte `mini-projeto/user-stories/` e `mini-projeto/requisitos/` para os templates.
