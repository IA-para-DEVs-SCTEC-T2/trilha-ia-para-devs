# Atividade - Semana 07

## Objetivo

Produzir documentacao tecnica de qualidade com apoio de IA e automatizar pelo menos uma tarefa do fluxo de desenvolvimento.

## O que o aluno deve produzir

- README completo para um projeto de sua escolha ou para o mini-projeto.
- Pelo menos um artefato de documentacao tecnica (ex: descricao de modulo, decisao arquitetural, guia de uso).
- Registro dos prompts utilizados para gerar a documentacao.
- Opcional: script ou configuracao de automacao de uma tarefa repetitiva.

## Como organizar a entrega

```
atividades/semana-07/
└── seu-nome/
    ├── docs/
    │   ├── README.md               # README do projeto documentado
    │   └── documentacao-tecnica.md # Artefato de documentacao
    ├── automacao/                  # Opcional: scripts de automacao
    ├── prompts-usados.md
    └── README.md
```

## Como registrar os prompts usados

Crie um arquivo `prompts-usados.md` com:

- O prompt enviado para gerar cada artefato de documentacao.
- O resultado obtido.
- O que foi aceito, ajustado ou descartado.

## Como versionar no Git

```bash
git checkout -b docs/atividade-semana-07-seu-nome
git add atividades/semana-07/seu-nome/
git commit -m "docs: adiciona documentacao semana 07 - seu-nome"
git push origin docs/atividade-semana-07-seu-nome
```

Abra um Pull Request apos o push.

## Criterios minimos de qualidade

- README claro, com objetivo, estrutura e instrucoes de uso.
- Documentacao tecnica objetiva e bem organizada.
- Prompts documentados com analise critica.
- Commits com mensagens descritivas.
- Sem informacoes inventadas ou imprecisas na documentacao.
