# Exemplos de Automacoes

Referencia pratica de automacoes comuns no desenvolvimento de software assistido por IA.

## Arquivos disponíveis

| Arquivo | Descricao |
|---------|-----------|
| `gerar-readme-com-ia.md` | Como gerar READMEs com apoio de IA |
| `gerar-issues-com-gh-cli.md` | Como criar Issues no GitHub via CLI |
| `revisar-pr-com-ia.md` | Como usar IA para revisar Pull Requests |
| `gerar-testes-com-ia.md` | Como gerar testes automatizados com IA |

## Quando usar automacoes

- Para tarefas repetitivas que seguem um padrao previsivel.
- Para acelerar a criacao de artefatos de documentacao.
- Para padronizar a criacao de Issues e PRs.
- Para gerar um ponto de partida que sera revisado e ajustado.

## Quando evitar automacoes

- Quando o contexto e muito especifico e a IA nao tem informacao suficiente.
- Quando o resultado nao sera revisado antes de ser publicado.
- Quando a automacao pode causar efeitos colaterais indesejados (ex: criar Issues em producao).
