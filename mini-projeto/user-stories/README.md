# User Stories

Esta pasta contem o modelo e exemplos de User Stories para o mini-projeto.

## O que e uma User Story

Uma User Story descreve uma necessidade do usuario de forma simples e centrada no valor que a funcionalidade entrega. Ela nao e uma tarefa tecnica, e uma descricao de comportamento esperado do sistema do ponto de vista de quem usa.

## Formato

```
Como [tipo de usuario],
quero [acao ou funcionalidade],
para [beneficio ou objetivo].
```

## Criterios de aceitacao em BDD

Os criterios de aceitacao descrevem o comportamento esperado do sistema em cenarios especificos:

```
Dado que [contexto inicial]
Quando [acao do usuario]
Entao [resultado esperado]
```

## Como usar

Use `modelo-user-story.md` como ponto de partida para cada User Story do seu projeto. Os exemplos do Shop4u sao apenas referencia didatica.
