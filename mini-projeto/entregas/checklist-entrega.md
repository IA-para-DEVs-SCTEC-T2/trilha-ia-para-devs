# Checklist de Entrega

Verifique cada item antes de considerar a entrega concluida.

## Documentacao

- [ ] `README.md` completo com descricao, tecnologias, instalacao e execucao
- [ ] `docs/PRD.md` preenchido
- [ ] User Stories com criterios de aceitacao em BDD
- [ ] Diagramas (UML ou C4) incluidos
- [ ] `docs/prompts.md` com registro dos prompts utilizados

## Repositorio

- [ ] Repositorio publico ou compartilhado com o professor
- [ ] Estrutura de pastas organizada
- [ ] Sem arquivos sensiveis commitados (.env, tokens, senhas)
- [ ] Historico de commits com mensagens descritivas
- [ ] Branches criadas a partir de Issues
- [ ] Pelo menos 1 Pull Request aberto e mergeado

## Codigo e testes

- [ ] Codigo funcional (pelo menos as funcionalidades principais)
- [ ] Testes automatizados presentes e passando
- [ ] Evidencias de execucao dos testes incluidas

## Entrega

- [ ] Links preenchidos em `links.md`
- [ ] Evidencias registradas em `evidencias.md`
- [ ] Relatorio de entrega preenchido (se solicitado)
- [ ] Apresentacao preparada
