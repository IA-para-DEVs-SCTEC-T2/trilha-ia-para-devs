# Criterios de Avaliacao

Os criterios estao organizados em tres niveis progressivos. Consulte tambem o arquivo `Avaliacao MINI PROJETO.pdf` para a rubrica completa.

---

## Nivel 1 - Iniciante

**Objetivo:** usar IA conversacional para criar os primeiros artefatos do projeto.

| Criterio | Descricao |
|----------|-----------|
| Diagrama tecnico | 1 diagrama UML ou C4 |
| User Stories com BDD | 3 User Stories com criterios de aceitacao |
| Wireframe ou mockup | Minimo 3 telas geradas com prompts |
| Estrutura de pastas | Gerada com IA e versionada no repositorio |
| `docs/PRD.md` | Visao geral, problema, objetivo, publico-alvo, arquitetura inicial e User Stories |
| `README.md` | Descricao, tecnologias, execucao e estrutura do repositorio |
| GitHub Flow | Branches e commits com convencao justificada |
| Testes automatizados | Minimo 2 testes |
| Registro de prompts | `docs/prompts.md` com os prompts utilizados |

---

## Nivel 2 - Intermediario

**Objetivo:** aplicar IA no fluxo de engenharia com backlog, GitHub Flow e documentacao organizada.

| Criterio | Descricao |
|----------|-----------|
| Diagrama tecnico | 1 diagrama UML ou C4 |
| User Stories com BDD | 3 User Stories com criterios de aceitacao |
| Wireframe ou mockup | Minimo 5 telas geradas com prompts |
| Estrutura de pastas | Gerada com IA e versionada |
| GitHub Issues | Issues derivadas de requisitos funcionais e documentacao inicial |
| `docs/PRD.md` | Visao geral, problema, objetivos, publico-alvo, regras de negocio, arquitetura inicial e User Stories |
| `README.md` | Descricao, tecnologias, execucao, estrutura de pastas, escolhas tecnicas e fluxo de desenvolvimento |
| Pull Request | Minimo 1 PR com template preenchido |
| GitHub Flow | Branches e commits seguindo convencao justificada |
| Testes automatizados | Minimo 3 testes |
| Registro de prompts | `docs/prompts.md` organizado |

---

## Nivel 3 - Avancado

**Objetivo:** demonstrar fluxo mais completo de engenharia de software assistida por IA, com rastreabilidade, documentacao e apresentacao tecnica.

| Criterio | Descricao |
|----------|-----------|
| Diagrama tecnico | 1 diagrama UML ou C4 com revisao tecnica documentada |
| User Stories com BDD | 3 User Stories com criterios de aceitacao e checklist tecnico |
| Wireframe e mockup | Minimo 5 telas com prompts |
| Estrutura de pastas | Gerada com IA, versionada e justificada |
| GitHub Issues | Issues derivadas de requisitos, organizadas com labels e rastreabilidade com PRs |
| `docs/PRD.md` | PRD completo: requisitos, regras, arquitetura, User Stories, fluxo de funcionamento, stack e proximos passos |
| `README.md` | Instalacao, execucao, estrutura, fluxo GitHub e convencoes |
| `CONTRIBUTING.md` | Padrao de contribuicao, branches, commits e PRs |
| Pull Request | Minimo 1 PR com template e referencia a Issue |
| GitHub Flow | Branch, commit, PR, merge e relacao com Issue |
| Testes automatizados | Minimo 5 testes |
| Registro de prompts | `docs/prompts.md` organizado |
| Apresentacao tecnica | Problema, solucao, arquitetura, uso de IA, testes e decisoes tecnicas |
