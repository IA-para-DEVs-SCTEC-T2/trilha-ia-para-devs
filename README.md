# Trilha IA para Devs

Material de apoio para a disciplina de Inteligencia Artificial aplicada ao desenvolvimento de software.

## Descricao

Este repositorio reune slides, exemplos praticos, atividades, templates e orientacoes para o mini-projeto avaliativo. O conteudo acompanha as semanas do modulo e serve como referencia durante e apos as aulas.

## Objetivo

Apoiar os alunos no desenvolvimento de habilidades praticas de engenharia de software assistida por IA, cobrindo desde engenharia de contexto e prompts ate testes automatizados, documentacao tecnica e automacao de fluxos com GitHub.

## Publico-alvo

Estudantes de desenvolvimento de software com conhecimento basico de programacao, Git e GitHub.

---

## Organizacao das pastas

```
trilha-ia-para-devs/
├── aulas/          # Materiais produzidos ou conduzidos pelo professor
├── atividades/     # Desafios e exercicios para os alunos resolverem
├── exemplos/       # Referencias praticas reutilizaveis
├── materiais/      # Guias, templates e referencias gerais
└── mini-projeto/   # Informacoes, criterios, requisitos e entregas do projeto avaliativo
```

### aulas/

Contem os slides em PDF, codigos demonstrados em aula e materiais de apoio organizados por semana. Os alunos devem usar essa pasta para revisar o conteudo apresentado pelo professor.

Semanas disponíveis:

- `semana-02-engenharia-contexto/` - Engenharia de contexto e prompts
- `semana-05-arquiteturas-mockups/` - Arquiteturas, mockups, User Stories e fluxo GitHub
- `semana-06-testes-com-ia/` - Testes automatizados com apoio de IA
- `semana-07-documentacao-automacao/` - Documentacao tecnica e automacao

### atividades/

Contem os enunciados e espacos para entrega das atividades semanais. Cada aluno deve criar sua propria branch, resolver a atividade e abrir um Pull Request.

### exemplos/

Contem arquivos de referencia pratica sobre automacoes, GitHub, Postman, prompts e testes. Nao sao entregas dos alunos, sao modelos para consulta e reutilizacao.

### materiais/

Contem guias de referencia, templates reutilizaveis, referencias bibliograficas e sugestoes de leitura. Util para consulta durante as atividades e o mini-projeto.

### mini-projeto/

Contem todas as informacoes sobre o projeto avaliativo: criterios de avaliacao, requisitos, user stories, diagramas, orientacoes de entrega e checklist.

---

## Fluxo recomendado para os alunos

1. Leia o README de cada semana em `aulas/` antes da aula.
2. Assista a aula e acompanhe os exemplos em `aulas/semana-XX/hands-on/`.
3. Consulte os arquivos em `exemplos/` para referencia pratica.
4. Resolva a atividade da semana em `atividades/semana-XX/`.
5. Use os templates em `materiais/templates/` para estruturar seus artefatos.
6. Aplique o aprendizado no mini-projeto em `mini-projeto/`.

---

## Uso de IA no desenvolvimento

- Use IA como ferramenta de apoio, nao como substituto do raciocinio.
- Sempre revise o que a IA gera antes de usar.
- Registre os prompts utilizados em cada atividade.
- Documente as decisoes tomadas com base nas sugestoes da IA.
- Consulte `exemplos/prompts/` para modelos de prompts reutilizaveis.
- Consulte `materiais/guias/guia-uso-de-ia-no-desenvolvimento.md` para boas praticas.

## Git e GitHub

- Nunca trabalhe diretamente na branch `main`.
- Crie uma branch para cada atividade ou funcionalidade.
- Faca commits pequenos e com mensagens descritivas.
- Abra Pull Requests para revisao antes de fazer merge.
- Consulte `exemplos/github/` para modelos de Issues, PRs e commits.

### Convencoes de branch

| Prefixo | Uso |
|---------|-----|
| `feat/` | Nova funcionalidade |
| `fix/` | Correcao de bug |
| `docs/` | Documentacao |
| `test/` | Testes |
| `chore/` | Tarefas de manutencao |

Exemplo: `feat/tela-login`, `docs/readme-semana-05`

### Convencoes de commit

```
feat: adiciona tela de login
fix: corrige validacao de senha vazia
docs: atualiza README com instrucoes de entrega
test: adiciona testes unitarios para calculo de frete
```

## Atividades e desafios

- Leia o enunciado completo antes de comecar.
- Registre os prompts usados em um arquivo `prompts-usados.md` dentro da sua entrega.
- Inclua evidencias de execucao (prints, logs ou saidas de terminal).
- Siga os criterios de qualidade descritos em cada atividade.

---

## Mini-projeto

O mini-projeto e o trabalho avaliativo do modulo. Consulte `mini-projeto/README.md` para entender o que deve ser entregue, como organizar o repositorio e como usar GitHub Issues, branches e Pull Requests no desenvolvimento.

Os criterios de avaliacao estao em `mini-projeto/criterios-avaliacao.md` e no arquivo `mini-projeto/Avaliacao MINI PROJETO.pdf`.

---

## Criterios gerais de qualidade

- Codigo funcional e testado.
- Documentacao clara e objetiva.
- Commits com mensagens descritivas.
- Uso consciente e registrado de IA.
- Evidencias de execucao incluidas nas entregas.
- Estrutura de repositorio organizada.
